# CRM 360 — Customer Intelligence System

![CRM 360 Dashboard Preview](Images/1.png) *(Note: You can replace this link with an actual screenshot of your dashboard later)*

**CRM 360** is a full-stack Customer Relationship Management (CRM) platform designed to help businesses track customer data, analyze sales KPIs, manage support tickets, and monitor web interactions in real time. 

Built with a focus on modern UI/UX (Glassmorphism) and a robust backend architecture, this project serves as a complete, enterprise-level application demonstrating end-to-end full-stack development.

---

## 🚀 Key Features

- **Role-Based Authentication:** Secure login system differentiating between `Admin` (full access) and `Agent` (view only) roles.
- **Dynamic Dashboard Analytics:** Real-time KPI aggregation (Total Revenue, Customer Growth, Pending Orders) powered by SQL.
- **Customer Management (CRUD):** Add new customers, view detailed 360° profiles (timelines, orders, tickets), and delete records.
- **Global Smart Search:** Instantly search across the entire database for customers by name, phone, or email.
- **Realistic Data Simulation:** Pre-configured with a Python database seeder that populates 1,900+ realistic records across 10 normalized tables.
- **Modern UI/UX:** Responsive, dark-themed Glassmorphism interface with smooth CSS animations and Chart.js integrations.

## 🛠️ Technology Stack

**Frontend:**

- JavaScript (ES6+, Fetch API)

**Backend:**
- Python 3
- Flask (REST API Server)
- Flask-CORS
- python-dotenv (Environment Variable Management)

**Database:**
- MySQL (10 Normalized Tables, Relational Schema)
- mysql-connector-python

---

## 🏗️ Architecture Overview

The system follows a classic Client-Server architecture:
1. **Client (Browser):** Vanilla JS frontend makes asynchronous `fetch()` calls to the backend.
2. **Server (Flask):** The Python backend exposes 20+ REST API endpoints (`app.py`), validating business logic and parsing JSON.
3. **Database (MySQL):** A relational database storing data across tables like `customer`, `orders`, `support_ticket`, and `call_log`.

## ⚙️ Local Setup Instructions

Follow these steps to run the project locally on your machine.

### 1. Prerequisites
- Python 3.10+
- MySQL Server & MySQL Workbench (or XAMPP)
- VS Code (with Live Server extension)

### 2. Database Setup
1. Open MySQL Workbench and create a new database:
   ```sql
   CREATE DATABASE crm360_db;
   ```
2. Run the provided SQL scripts in the `Database/` folder to create the 10 tables.

### 3. Backend Configuration
1. Open your terminal and navigate to the `Backend/` folder.
2. Install the required Python packages:
   ```bash
   pip install -r requirements.txt
   ```
3. Create a `.env` file in the `Backend/` folder and add your MySQL credentials:
   ```env
   DB_HOST=localhost
   DB_USER=root
   DB_PASSWORD=your_mysql_password_here
   DB_NAME=crm360_db
   ```
4. Generate the database seed data and user accounts:
   ```bash
   python create_users.py
   python seed_data.py
   ```
5. Start the Flask API server:
   ```bash
   python app.py
   ```
   *(The server will run on `http://localhost:5000`)*

### 4. Frontend Launch
1. Open the project folder in VS Code.
2. Right-click on `Frontend/login.html` and select **"Open with Live Server"**.
3. Log in using the admin credentials generated by `create_users.py`:
   - **Username:** `admin`
   - **Password:** `admin123`
